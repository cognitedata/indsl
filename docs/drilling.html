Rotation Detection

indsl.drilling.rotation_detection(rot_vel: Series, thresh: float = 0) → Series

Rotation detection.

A simple on/off detection of rotation of a drill string based on a threshold

Parameters:

  • rot_vel – Rotational velocity [rpm]. Time series with the rotational velocity of the drill string

  • thresh – Rotation threshold [rpm]. Minimum rotation value for the drill string to be considered rotating

Returns:

Rotation periods. Binary time series indicating rotation on or rotation off: Rotation on = 1, Rotation off = 0.

Return type:

pandas.Series

On-Bottom Detection

indsl.drilling.onbottom_detection(bit_depth: Series, hole_depth: Series, thresh: float = 0) → Series

On bottom detection.

A simple on/off bottom of the hole detection for a drilling assembly

Parameters:

  • bit_depth – Measured depth of the drill bit. Time series with the measured depth of the drilling string

  • hole_depth – Measured depth of the well. Time series with the measured depth of the well

  • thresh – Onbottom threshold. Minimum distance between hole depth and bit depth for the drill bit to be considered off bottom

Returns:

On bottom periods. Binary time series indicating if the drill bit is on bottom or off bottom: On bottom = 1, Off bottom = 0.

Return type:

pandas.Series

Raises:

UserValueError – Raises an error if either hole depth or bit depth is all nan values, or if bit depth > hole depth

Drilling Performance Metrics

Mechanical Specific Energy

indsl.drilling.mse(torque: Series, rpm: Series, wob: Series, rop: Series, bs: Series) → Series

Mechanical Specific Energy.

Calculates the Mechanical Specific Energy (MSE) for drilling operations. MSE is a measure of the energy required to remove a unit volume of rock and is used to evaluate drilling efficiency.

The formula for MSE in metric units:

\[\mathrm{MSE} = \frac{T \cdot \omega + \mathrm{WOB} \cdot \mathrm{ROP}}{A \cdot \mathrm{ROP}}\]

Where: - \(T\) is the torque [N.m] - \(\omega\) is the angular velocity [rad/s] = \(\mathrm{RPM} \cdot \frac{2\pi}{60}\) - \(\mathrm{WOB}\) is the weight on bit [N] - \(\mathrm{ROP}\) is the rate of penetration [\(\mathrm{m/h}\)] - \(A\) is the bit area [\(\mathrm{m^2}\)]

This can be simplified to:

\[\mathrm{MSE} = \frac{T \cdot \mathrm{RPM} \cdot \frac{2\pi}{60} + \mathrm{WOB} \cdot \mathrm{ROP}}{A \cdot \mathrm{ROP}}\]

For more information on Mechanical Specific Energy derivation, see: Mechanical Specific Energy Derivation.

Parameters:

  • torque – Torque [N.m]. Time series with the torque applied to the drill string.

  • rpm – Rotational velocity [rpm]. Time series with the rotational velocity of the drill string in revolutions per minute.

  • wob – Weight on bit [N]. Time series with the weight applied to the drill bit in Newtons.

  • rop – Rate of penetration [\(\mathrm{m/h}\)]. Time series with the rate of penetration in meters per hour.

  • bs – Bit size area [\(\mathrm{m^2}\)]. Time series with the bit size in [m2]

Returns:

Mechanical Specific Energy [\(\mathrm{J/m^3}\) or \(\mathrm{Pa}\)].

Time series with the calculated MSE values. MSE represents the energy required to remove a unit volume of rock. Returns NaN values where any input is NaN or where division by zero would occur (e.g., when bit area or ROP is zero or negative).

Return type:

pandas.Series

Depth Of Cut

indsl.drilling.doc(rop: Series, rpm: Series) → Series

Depth Of Cut.

Calculates the Depth Of Cut (DOC) for drilling operations. DOC represents the distance the drill bit advances per revolution and is a key parameter for evaluating drilling performance and bit efficiency.

The formula for DOC in metric units:

\[\mathrm{DOC} = \frac{\mathrm{ROP}}{\mathrm{RPM}}\]

Where: - \(\mathrm{ROP}\) is the rate of penetration [\(\mathrm{m/h}\)] - \(\mathrm{RPM}\) is the rotational velocity [rpm]

The result is in meters per revolution [\(\mathrm{m/rev}\)].

Parameters:

  • rop – Rate of penetration [\(\mathrm{m/h}\)]. Time series with the Rate Of Penetration in [m/hr]

  • rpm – Rotation speed [rpm]. Time series with Rotation speed of the drill string in [rpm]

Returns:

Depth Of Cut [\(\mathrm{m/rev}\)].

Time series with the calculated DOC values. DOC represents the distance the drill bit advances per revolution. Returns NaN values where any input is NaN or where division by zero would occur (e.g., when RPM is zero or negative).

Return type:

pandas.Series

Drilling State Analysis

State Stand

indsl.drilling.state_stand(time: Series, hkld: Series, dmd: Series, dbit: Series, bpos: Series, rpm: Series, tors: Series, spp: Series, flow: Series) → DataFrame

Drilling state stand detection.

Detects drilling operational states at the stand level and calculates block velocity based on multiple drilling parameters. The function classifies drilling activities into different state modes and computes the block velocity from block position changes over time.

Block velocity is calculated as the time derivative of block position:

\[\mathrm{bvel} = \frac{d(\mathrm{bpos})}{dt}\]

Where: - \(\mathrm{bpos}\) is the block position [m] - \(t\) is time [ms] - \(\mathrm{bvel}\) is the block velocity [\(\mathrm{m/s}\)]

The state_stand is an integer code (0-8) representing drilling operational states at the stand level. See StateStand enum for the complete list of state codes and descriptions.

The state_mode is an integer code (0-15) representing detailed drilling operational modes. See StateMode enum for the complete list of mode codes and descriptions.

Parameters:

  • time – Time [ms]. Time series in milliseconds since 1970-01-01 (Unix epoch). Used for temporal analysis and time-based calculations.

  • hkld – HookLoad [kN]. Time series with hookload values. Used to detect slips (very low HookLoad) and drilling activity.

  • dmd – HoleDepth [m]. Time series with hole depth values. Used to detect drilling progress and connection intervals.

  • dbit – BitDepth [m]. Time series with bit depth values. Used to detect when bit is on-bottom vs off-bottom.

  • bpos – BlockPosition [m]. Time series with block position values. Used to detect pipe movement during connections.

  • rpm – Rotary Speed [rpm]. Time series with rotary speed values. Used to classify rotating vs non-rotating states.

  • tors – SurfaceTorque [kN.m]. Time series with surface torque values. Used to detect rotation and drilling activity.

  • spp – StandpipePressure [kPa]. Time series with standpipe pressure values. Used to detect pumping activity.

  • flow – FlowIn [\(\mathrm{L/min}\)]. Time series with flow-in values. Used to detect pumping activity.

Returns:

Drilling state results.

DataFrame with three columns: - bvel: Block velocity [\(\mathrm{m/s}\)] calculated from block position changes over time.

Returns NaN if insufficient data points (< 2) are provided.

  • state_stand: Integer code representing the drilling operational state at stand level (0-8).

    Returns ST_ABSENT (0) if insufficient data points are provided.

  • state_mode: Integer code representing the detailed drilling operational mode (0-15).

    Returns MD_ABSENT (0) if insufficient data points are provided.

Return type:

pandas.DataFrame
